# XR Spatial Interface Design

Use this for VR, AR, MR, WebXR, 3D web, spatial computing, immersive dashboards, spatial training, avatars, and object-based interfaces.

## Steps

1. Define XR context.
   - Device class, environment, session length, input modalities, user posture, safety constraints.
   - Verify: physical context and input method are explicit.
2. Map spatial objects.
   - Objects, attributes, actions, relationships, states, ownership, and anchors.
   - Verify: at least one object map exists before panel design.
3. Design interaction model.
   - Gaze, pinch, hand, controller, voice, haptics, touch, keyboard, or fallback.
   - Verify: every primary action has feedback and undo/recovery.
4. Design comfort.
   - Locomotion, scale, distance, field of view, duration, pace, onboarding, pause/recenter.
   - Verify: motion sickness and fatigue risks are addressed.
5. Prototype.
   - Choose storyboard, graybox, 3D mock, WebXR prototype, or engine prototype.
   - Verify: prototype can test the riskiest assumption.
6. Validate.
   - Observe users, capture task metrics, comfort feedback, spatial analytics, and failure points.
   - Verify: validation includes both task success and physical comfort.

## XR Output Checklist

- Spatial object map.
- Input modality matrix.
- Comfort/safety checklist.
- Onboarding sequence.
- Accessibility alternatives.
- Analytics events.
- Prototype fidelity and known limitations.
